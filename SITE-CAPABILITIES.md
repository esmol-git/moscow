# Возможности шаблона и план для сайта недвижимости

## Структура шаблона (flsStart4)

- **Точка входа:** `src/index.html` → подключает `main.html`, блоки header / main / footer.
- **Страницы:** `src/components/pages/<name>/` — index, при необходимости новые страницы через `newpage` (шаблон копирует из main).
- **Стили:** `src/styles/` — settings, variables, style.scss; у каждого компонента свой `*.scss`.
- **Скрипты:** `src/js/app.js` — общий вход; компоненты подключают себя через импорт (header → menu, scroll; app → scrollto, watcher).
- **Алиасы:** `@components`, `@styles`, `@js`, `@img`, `@fonts` и др. в `template.config.js`.

## Что уже заложено под лендинг недвижимости

| Элемент | Где | Назначение |
|--------|-----|------------|
| Палитра цветов | `variables.css`, `settings.scss` | --re-dark, --re-steel, --re-dust, --re-sky, --re-light |
| Фиксированный хедер | header + `data-fls-header-scroll` | При скролле вниз скрывается, вверх — показывается |
| Плавная прокрутка | scrollto + watcher | Якоря, учёт высоты хедера, подсветка пункта меню |
| Мобильное меню | menu (data-fls-menu) | Бургер, выезд панели, оверлей, закрытие по клику снаружи и по ссылке |
| Секции лендинга | index: hero, objects, about, contacts | Блоки с data-fls-watcher="navigator" для навигации |

## Компоненты шаблона, полезные для сайта недвижимости

### Layout (уже используем / логично добавить)

| Компонент | Путь | Для чего на сайте |
|-----------|------|-------------------|
| **header** | `layout/header/` | Уже: лого, меню, фикс при скролле |
| **menu** | `layout/menu/` | Уже: десктоп + мобильное меню с якорями |
| **footer** | `layout/footer/` | Ссылки, контакты, копирайт |
| **slider** | `layout/slider/` | Слайдер героя, карусель объектов (Swiper) |
| **gallery** | `layout/gallery/` | Lightgallery — галерея фото объекта (модалка, зум) |
| **map** | `layout/map/` | Карта с маркером офиса/объекта (Leaflet) |
| **popup** | `layout/popup/` | Модалки: заявка, просмотр объекта, обратный звонок |
| **spollers** | `layout/spollers/` | FAQ, «Подробнее об объекте», аккордеон |
| **tabs** | `layout/tabs/` | Табы: описание / планировки / документы по объекту |
| **showmore** | `layout/showmore/` | «Показать ещё» для списка объектов |
| **grid** | `layout/grid/` | Сетка карточек объектов |
| **masonry** | `layout/masonry/` | Плитка разной высоты (фото объектов) |
| **beforeafter** | `layout/beforeafter/` | До/после ремонта (опционально) |

### Формы

| Компонент | Путь | Для чего |
|-----------|------|----------|
| **form** | `forms/form/` | Отправка заявки (data-fls-form, валидация, popup после отправки) |
| **input** | `forms/input/` | Маска телефона, авто-высота textarea, показ пароля |
| **button** | `forms/button/` | Кнопки отправки и CTA |
| **select** | `forms/select/` | Город, тип объекта, комнаты и т.п. |
| **checkbox / radio** | `forms/checkbox/`, `forms/radio/` | Согласие с политикой, фильтры |
| **datepicker** | `forms/datepicker/` | Дата просмотра (если нужна) |

### Эффекты (по желанию)

| Компонент | Назначение |
|-----------|------------|
| **watcher** | Уже: появление блоков при скролле (анимация секций) |
| **scrollto** | Уже: якоря и навигация |
| **parallax** | Параллакс в герое или блоках |
| **gsap** | Сложная анимация текста/блоков |
| **preloader** | Прелоадер при первой загрузке |
| **tippy** | Подсказки (иконки, фильтры) |
| **zoom** | Увеличение по клику на превью |

## Рекомендуемый порядок внедрения

1. **Карточки объектов** — секция «Объекты»: grid или своя сетка, ссылка в popup или отдельная страница.
2. **Галерея в модалке** — в карточке/попапе: gallery (Lightgallery) для фото объекта.
3. **Форма заявки** — в footer или popup: form + input (имя, телефон, маска) + отправка в popup.
4. **Карта** — блок «Контакты»: map с маркером офиса.
5. **Слайдер** — в hero или блок «Акции/новые объекты»: slider (Swiper).
6. **FAQ / спойлеры** — spollers внизу или в блоке «О нас».
7. **Табы** — на странице объекта: описание / планировки / документы (tabs).

## Настройки template.config.js, которые могут понадобиться

- **php.enable: true** — если форма будет отправляться через PHP (sendmail).
- **images.optimize** — уже включено (webp, размеры); для объектов важен `attrignore` для фото, которые не резать.
- **fonts.download** — подключить шрифты из макета при необходимости.
